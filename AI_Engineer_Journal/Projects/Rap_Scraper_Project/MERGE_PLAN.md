# 🔄 План объединения баз данных

## 📊 Текущая ситуация

У нас есть **два сценария**:

### 🔄 Сценарий 1: Продолжить скрапинг + объединить позже (РЕКОМЕНДУЮ)
✅ **Преимущества:**
- Не теряем время на сбор данных
- Параллельная работа двух процессов
- SQLite отлично подходит для merge операций

### 🛑 Сценарий 2: Остановить скрапинг
❌ **Недостатки:**
- Потеря времени
- Возможная потеря API квот

## 🛠️ Что я создал для объединения

### 1. **merge_databases.py** - основной скрипт объединения
```bash
python merge_databases.py
```
**Что делает:**
- Создает резервные копии обеих БД
- Объединяет записи без дубликатов
- Поддерживает как старую, так и новую схему
- Показывает детальную статистику

### 2. **rap_scraper_optimized.py** - улучшенный скрапер
**Ключевые улучшения:**
- ✅ Мониторинг памяти (psutil)
- ✅ Генераторы вместо списков
- ✅ Автоматическая очистка памяти
- ✅ Расширенные метаданные
- ✅ WAL режим SQLite
- ✅ Батч-коммиты по 1000 записей
- ✅ Оценка качества текстов

### 3. **migrate_database.py** - миграция существующей БД
```bash
python migrate_database.py
```
**Добавляет новые поля:**
- genre, release_date, album
- language, explicit, song_art_url
- popularity_score, lyrics_quality_score

## 📋 Пошаговый план действий

### Когда ваш скрапинг на другом компьютере завершится:

1. **Скопируйте файл базы данных**
   ```bash
   # Скопируйте rap_lyrics.db с другого компьютера
   # Переименуйте в rap_lyrics_remote.db
   ```

2. **Запустите объединение**
   ```bash
   python merge_databases.py
   ```

3. **Замените старую БД на новую**
   ```bash
   # Переименуйте результат в rap_lyrics.db
   ```

4. **Используйте новый оптимизированный скрапер**
   ```bash
   python rap_scraper_optimized.py
   ```

## 🔧 Настройки для объединения

В `merge_databases.py` можно настроить:
```python
# Пути к файлам
old_db = "rap_lyrics.db"          # Ваша текущая БД
new_db = "rap_lyrics_remote.db"   # БД с другого компьютера
```

## ❓ FAQ

**Q: Безопасно ли объединять БД?**
A: Да, скрипт создает резервные копии автоматически

**Q: Что если будут дубликаты?**
A: Скрипт использует `INSERT OR IGNORE` по URL и genius_id

**Q: Нужно ли останавливать текущий скрапинг?**
A: Нет! Можете продолжать, а потом объединить

**Q: Сколько времени займет объединение?**
A: Для 44k записей ~ 2-5 минут
