# Мировоззрение в AI/ML: Ключевые концепции из доклада Карпатого

## Основные концепции докладе

### 1. Эволюция программного обеспечения (Software 1.0 → 2.0 → 3.0)

**Software 1.0**: Традиционный код, который мы пишем для компьютеров
- Python, JavaScript, C++ и другие языки программирования
- Явные инструкции и алгоритмы

**Software 2.0**: Веса нейронных сетей
- Параметры обучаются на данных через оптимизацию
- Hugging Face как "GitHub для нейросетей"
- Пример: модели компьютерного зрения, распознавания речи

**Software 3.0**: Промпты для LLM
- **Революционно**: программирование на естественном языке (английском)
- LLM как программируемые компьютеры
- Промпты = программы для LLM

### 2. LLM как новые операционные системы

**Аналогии с коммунальными службами:**
- LLM лабы тратят capex на обучение, opex на обслуживание
- Оплата за токены = оплата за электричество
- "Интеллектуальные блэкауты" при падении сервисов

**Аналогии с операционными системами:**
- Контекстное окно = оперативная память
- LLM = CPU для обработки языка
- Закрытые (GPT, Claude) vs открытые (Llama) экосистемы
- Мы находимся в "1960-х годах LLM-компьютинга"

### 3. Психология LLM: "Духи людей"

**Суперспособности:**
- Энциклопедическая память
- Мгновенная обработка огромных объемов текста

**Когнитивные дефициты:**
- Галлюцинации и выдуманная информация
- "Рваный интеллект": сверхчеловеческий в одном, детские ошибки в другом
- Антероградная амнезия: не накапливают опыт между сессиями
- Уязвимость к prompt injection

### 4. Приложения с частичной автономией

**Ключевые принципы успешных LLM-приложений:**
- **Управление контекстом**: автоматическая подача релевантной информации
- **Оркестрация множественных вызовов**: embedding модели + чат модели + специализированные модели
- **GUI для аудита**: визуальные дифы вместо текста
- **Слайдер автономии**: от автодополнения до полной автономии

**Примеры:**
- Cursor: различные уровни помощи в кодинге
- Perplexity: быстрый поиск vs глубокое исследование

### 5. Человеко-ИИ сотрудничество

**Золотое правило**: Люди генерируют → ИИ верифицирует
- **Ускорение верификации**: GUI, визуализации
- **Контроль ИИ**: избегать слишком больших изменений
- Инкрементальный подход вместо массивных автономных изменений

### 6. Vibe Coding: Демократизация программирования

- Теперь каждый может программировать на естественном языке
- Дети могут создавать приложения без изучения синтаксиса
- Барьеры входа упали с 5-10 лет обучения до нескольких часов

### 7. Инфраструктура для агентов

**Адаптация под ИИ:**
- `llm.txt` файлы (аналог `robots.txt`)
- Документация в Markdown вместо HTML
- Замена "нажмите" на API команды (curl)
- Model Context Protocol (MCP) от Anthropic

## Что устарело к 2025 году

### Технологические изменения:
1. **Контекстные окна значительно выросли**: от ~100K токенов до миллионов
2. **Появились новые модели**: Claude 4, GPT-o1, Gemini 2.0
3. **Мультимодальность стала стандартом**: видео, аудио, изображения
4. **Агенты стали более надежными**: лучше работают автономно

### Устаревшие ограничения:
1. **Проблемы с математикой почти решены** благодаря reasoning моделям
2. **Tool use стал намного надежнее**
3. **Появились модели с persistent memory**

## Самое важное для запоминания

### 🎯 Фундаментальные инсайты:

1. **Мы переживаем третью революцию в программировании** - после появления языков высокого уровня и ООП
2. **LLM = новые операционные системы**, а не просто инструменты
3. **Программирование на естественном языке демократизирует разработку**
4. **Частичная автономия > полная автономия** для практических применений
5. **Успех = быстрый цикл генерации-верификации** между человеком и ИИ

### 🛠️ Практические принципы:

1. **Всегда держите ИИ "на поводке"** - контролируйте размер изменений
2. **GUI критически важен** для быстрого аудита работы ИИ
3. **Инкрементальный подход** работает лучше больших автономных задач
4. **Контекст-менеджмент** - ключевая функция LLM-приложений
5. **Слайдер автономии** должен быть в каждом ИИ-продукте

### 🚀 Возможности для бизнеса:

1. **Любое ПО может стать частично автономным**
2. **Переписывание документации под ИИ** - новая необходимость
3. **"Встреча ИИ на полпути"** - адаптация API и интерфейсов
4. **Vibe coding** открывает новые рынки непрограммистов

## Мои дополнения (на основе текущего состояния AI в 2025)

### Что произошло после доклада:

1. **Reasoning модели (o1, o3)** кардинально улучшили способности к математике и логике
2. **Компьютерные агенты** (Claude Computer Use) стали реальностью
3. **Мультимодальные модели** стали mainstream - теперь это не про будущее, а про сегодня
4. **Персональные ИИ-ассистенты** начали появляться на устройствах (Llama на телефонах)
5. **ИИ-инженеры** стали отдельной профессией

### Новые тренды:

1. **Agentic AI workflows** - цепочки специализированных агентов
2. **RAG + Fine-tuning гибриды** для специфических доменов
3. **ИИ в продакшене** требует новой инженерной культуры (prompt versioning, A/B testing промптов)
4. **Edge AI** - модели начинают работать локально
5. **Синтетические данные** как основа обучения новых моделей

### Практический совет:
Не гонитесь за каждой новой моделью. Сосредоточьтесь на понимании фундаментальных принципов работы с LLM и создании reliable продуктов. Технологии меняются быстро, но принципы остаются.
