# ТЕХНИЧЕСКОЕ ЗАДАНИЕ ДЛЯ CLAUDE AGENT: Интеграция LangChain в Genius Scraper

## 🎯 ЦЕЛЬ ПРОЕКТА
Улучшить существующий scraper Genius.com с помощью LangChain для создания высококачественного датасета для обучения модели генерации "живых" (не ИИшных) музыкальных треков.

## 📊 ТЕКУЩЕЕ СОСТОЯНИЕ
- ✅ Работающий scraper Genius.com
- ✅ Собрано ~10,000 песен  
- 🎯 Цель: 50,000 качественных треков
- 🤖 Планируется: обучение ML модели для генерации музыки

## 🔧 ЗАДАЧИ ДЛЯ РЕАЛИЗАЦИИ

### ЭТАП 1: Настройка LangChain (СРОЧНО)
1. **Установить зависимости:**
   ```bash
   pip install langchain langchain-openai langchain-community pydantic
   ```

2. **Создать Pydantic модели для структурированных данных:**
   - `SongMetadata` - базовые метаданные (жанр, год, исполнитель, настроение)
   - `LyricsAnalysis` - анализ текста (структура, сложность, темы, рифмовка)
   - `QualityMetrics` - оценка "живости" трека vs ИИ-генерации

3. **Интегрировать LangChain в существующий scraper:**
   - Добавить AI-анализ к каждой скрапленной песне
   - Сохранять обогащенные данные в JSONL формате
   - Добавить валидацию данных через Pydantic

### ЭТАП 2: Обогащение данных (ПРИОРИТЕТ)
1. **Обработать существующие 10k песен:**
   - Прогнать через LangChain для извлечения метаданных
   - Классифицировать по жанрам и настроению
   - Оценить структуру и сложность текстов

2. **Создать pipeline для новых 40k песен:**
   - Автоматический AI-анализ при скрапинге
   - Фильтрация низкокачественных треков
   - Детекция дублей и плагиата

### ЭТАП 3: Подготовка к ML (БУДУЩЕЕ)
1. **Создать обучающий датасет:**
   - Разделение train/validation/test (80/10/10)
   - Группировка по жанрам и годам
   - Создание признаков для ML модели

2. **Анализ паттернов "живой" музыки:**
   - Выявление что делает треки популярными
   - Анализ эволюции стилей по годам
   - Создание "рецептов" для разных жанров

## 🛠️ ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### Архитектура:
```
Existing Scraper → LangChain Enhancer → Structured Data → ML Pipeline
```

### Необходимые компоненты:
- **OpenAI API** для анализа (альтернатива: локальные модели)
- **Pandas** для обработки данных
- **JSONL** для хранения обогащенных данных
- **Pydantic** для валидации структуры

