# 📱 Инструкция для ноутбука (где идет скрапинг)

## 🎯 **После завершения скрапинга на ноутбуке:**

### 1. **Обновить проект с GitHub**
```bash
git pull origin master
```

### 2. **Установить новую зависимость**
```bash
pip install psutil>=5.9.0
```

### 3. **Скопировать базу данных на основной компьютер**
Способы передачи `rap_lyrics.db`:
- USB флешка
- Cloud (Google Drive, OneDrive)
- Локальная сеть
- Email (если размер позволяет)

### 4. **Переименовать файлы для объединения**
На основном компьютере:
```bash
# Переименуйте БД с ноутбука
rename "rap_lyrics.db" "rap_lyrics_laptop.db"
```

### 5. **Запустить объединение баз данных**
```bash
python merge_databases.py
```

Скрипт автоматически:
- ✅ Найдет обе базы данных
- ✅ Создаст резервные копии
- ✅ Объединит без дубликатов
- ✅ Покажет статистику

### 6. **Использовать новый оптимизированный скрапер**
```bash
python rap_scraper_optimized.py
```

## 📊 **Что получите после объединения:**

- 🎵 Все уникальные песни из обеих БД
- 📈 Мониторинг памяти и ресурсов
- 🏷️ Метаданные (жанр, год, качество)
- 🚀 Улучшенную производительность
- 💾 Автоматические резервные копии

## ❓ **Если возникнут проблемы:**

1. **Проверьте размер БД**: `ls -lh rap_lyrics.db`
2. **Проверьте статистику**: `python check_db.py`
3. **Тест нового скрапера**: `python test_optimized_scraper.py`

## 🚨 **Важно:**
- Резервные копии создаются автоматически
- Дубликаты исключаются по URL и genius_id
- Новый скрапер использует меньше памяти
- WAL режим увеличивает скорость записи
